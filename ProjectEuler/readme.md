1. 时间复杂度
   1. 意义：在算法数据规模发生变化时，算法的执行时间如何变化。
   2. 误区：不是算法的执行时间。
   3. 从小到大的常见时间复杂度排序：
      1. 常数级：O(1)，几条语句
      2. 对数级O(logn)，二分查找，log的底数不重要(对数的换底公式)
      3. 线性：O(n)
      4. 线性对数：O(nlogn)，快速排序
      5. 平方/立方级：O(n^2)冒泡 O(n^3)树的遍历
   4. 原则：抓大放小
      1. 顺序执行的两段代码时间复杂度是相加的关系；
      2. 相加后只保留最大项
      3. 例如：O(nlogn + n^2)，只保留O(n^2)
   5. 方法：见例子。
2. 空间复杂度
   1. 意义：在算法数据规模发生变化时，消耗的内存空间如何变化。
   3. 从小到大的常见时间复杂度排序：
      1. 常数级：O(1)，几个变量
      2. 对数级O(logn)，递归的深度
      3. 线性：O(n)，一维数组长度为n
      4. 平方/立方级：O(n^2)，二维复杂度
